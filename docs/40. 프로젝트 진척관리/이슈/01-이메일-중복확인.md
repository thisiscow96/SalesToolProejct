# 이슈 1 — 이메일 중복 확인 — 처음 입력한 이메일도 "이미 사용 중"

**등록날짜:** 2025-03-13

---

- **무슨 이슈였는지 (현상)**  
  DB에 해당 이메일이 없는데도 "이미 사용 중인 이메일입니다"로 표시됨.
- **이슈 원인**  
  프론트: 서버 오류(500, 네트워크 오류) 시에도 `data.available === false`이면 무조건 "이미 사용 중"으로 표시. 백엔드: 500/에러 응답에 사용자용 메시지 없어 원인 구분 불가.
- **조치 방법**  
  `!res.ok`일 때는 "확인할 수 없습니다" 등으로 표시하고, "이미 사용 중"은 **200 응답이면서 `available === false`일 때만** 표시. 백엔드 500 응답에 `message`, 필요 시 `error` 필드 포함.
- **교훈**  
  에러 응답과 정상 응답을 구분하고, 실패 시 사용자 메시지를 명확히 할 것.
