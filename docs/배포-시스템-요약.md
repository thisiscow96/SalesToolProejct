# 배포·시스템 요약 (원래 계획 vs 현재)

## 원래 사용하려 했던 시스템

| 구분 | 서비스 | 역할 |
|------|--------|------|
| **백엔드 호스팅** | Railway | Node(Express) + PostgreSQL 한 플랫폼에서 실행, GitHub 푸시 시 자동 배포 |
| **DB** | PostgreSQL (Railway) | Railway 프로젝트 안에서 Postgres 서비스로 제공 |
| **프론트 호스팅** | Vercel | React(Vite) 빌드 결과물 정적 호스팅, 사용자 접속 URL 제공 |
| **이메일** | Gmail SMTP | 회원가입 시 인증번호 발송 (nodemailer + smtp.gmail.com:587) |
| **버전/배포 연동** | GitHub | 코드 저장소, Railway·Vercel과 연결해 자동 배포 |

---

## 현재 추가·변경된 것 (요약)

| 구분 | 변경 내용 | 이유 |
|------|-----------|------|
| **백엔드 호스팅** | **Render** 추가 (대안) | Railway는 아웃바운드 SMTP(587/465)를 막아서 Gmail 발송 시 **ETIMEDOUT**. Gmail SMTP를 쓰려면 Render로 백엔드를 옮김. |
| **DB** | **Postgres (Render)** | Render에서 Web Service 쓸 때, 같은 Render 대시보드에서 Postgres 서비스를 만들고 **Internal Database URL** 로 연결. |
| **프론트** | 변경 없음 | 계속 Vercel. 다만 API 주소만 Railway → **Render URL** 로 바꿔서 연동. |
| **이메일** | 변경 없음 | 계속 Gmail SMTP. 단, **Railway 위에서는 발송 불가**, **Render 위에서는 발송 가능**. |

---

## 한 줄 정리

- **원래:** Railway(백엔드+DB) + Vercel(프론트) + Gmail SMTP  
- **현재:** Gmail 인증 메일을 쓰려면 백엔드+DB를 **Render**로 옮기고, 프론트(Vercel)는 **Render 백엔드 URL**로 API 호출하도록 설정.

Railway는 그대로 두고 프론트만 쓰면 이메일 인증은 안 되고, Render로 백엔드 이전하면 Gmail SMTP 그대로 사용 가능.
