# Solution

본 프로젝트는 기존 문제를 해결하기 위해 기능 단위 접근이 아닌, **업무 흐름 중심의 구조 재설계**를 기준으로 진행했습니다.

## Before

- 매입/매출/수금 데이터가 각각 분산 관리됨
- 동일 데이터 반복 입력 발생
- 변경 이력 관리 불가
- 커뮤니케이션 의존도가 높음

## After

- 재고·매입·매출·수금·폐기 흐름을 하나의 체계로 통합
- 입력 단계 최소화 및 사용자 중심 UI 구조 적용
- 변경 이력 추적 가능한 운영 구조 구축
- 데이터 기반으로 업무 흐름 관리 가능

## 설계 방향

### 1. 업무 흐름 중심 구조 설계

단순 데이터 저장이 아닌, 실제 업무 진행 순서를 기준으로 시스템을 구성했습니다.

### 2. 데이터 정합성 확보

각 단계별 데이터가 단절되지 않고 연결되도록 구조를 설계했습니다.

### 3. 운영 가능한 시스템 지향

배포, 문서화, 변경 이력 관리까지 포함하여 지속적으로 운영 가능한 구조를 고려했습니다.
