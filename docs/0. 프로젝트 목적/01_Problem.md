# Problem

기존 판매 관리 시스템은 실제 업무 환경에서 다음과 같은 문제가 반복적으로 발생하고 있었습니다.

## 1. 반복 입력 구조

- 매입, 매출, 수금 등 각 단계에서 동일 데이터를 반복 입력
- 입력 과정이 길어지면서 작업 시간 증가 및 입력 오류 발생

## 2. 데이터 분산 관리

- 매입/매출/수금/폐기 이력이 각각 분리되어 관리됨
- 전체 흐름을 한 번에 파악하기 어려워 추적성이 낮음

## 3. 운영 이력 관리 부재

- 변경 사항에 대한 기록이 남지 않아 유지보수 비효율 발생
- 문제 발생 시 원인 추적이 어려움

## 4. 커뮤니케이션 의존 구조

- 시스템이 아닌 사람 간 커뮤니케이션에 의존하여 업무 진행
- 전달 오류 및 누락 리스크 존재

이러한 문제로 인해 업무 효율 저하와 운영 리스크가 지속적으로 발생하는 상황이었습니다.
