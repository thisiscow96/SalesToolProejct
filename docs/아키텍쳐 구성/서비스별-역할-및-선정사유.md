# 서비스별 역할 및 선정 사유

## 1. 개발·버전 관리

### Git
| 항목 | 내용 |
|------|------|
| **역할** | 로컬에서 소스 코드 버전 관리. 커밋·브랜치로 변경 이력을 보관하고, 원격 저장소와 동기화하는 기준이 됨. |
| **선정 사유** | 업계 표준 버전 관리 도구이며 무료. 로컬에서 히스토리 관리와 롤백이 가능하고, GitHub·Railway 등과 자연스럽게 연동됨. |

### GitHub
| 항목 | 내용 |
|------|------|
| **역할** | 원격 Git 저장소 호스팅. 코드 백업·공유, 팀 협업, CI/CD·배포(예: Railway)와 레포지토리 연결에 사용. |
| **선정 사유** | 무료 플랜으로 프라이빗/퍼블릭 레포 사용 가능. Railway·Vercel 등 대부분의 배포·연동 서비스가 GitHub 연동을 지원해 배포 파이프라인 구성이 쉬움. |

---

## 2. 런타임·패키지 관리

### Node.js
| 항목 | 내용 |
|------|------|
| **역할** | JavaScript 런타임. 백엔드(Express) 서버 실행과 프론트엔드(Vite) 빌드·개발 서버 실행에 사용. |
| **선정 사유** | 프론트·백엔드 모두 JavaScript로 통일 가능. LTS 버전으로 안정적이며, 설치·배포 환경(Railway 등)에서 널리 지원됨. |

### npm
| 항목 | 내용 |
|------|------|
| **역할** | Node 패키지 설치·의존성 관리. `package.json` 기반으로 `server/`, `web/` 의 라이브러리 설치 및 스크립트(`npm start`, `npm run build` 등) 실행. |
| **선정 사유** | Node.js 설치 시 기본 포함되어 별도 도입 비용 없음. 대부분의 Node 호스팅이 `npm install`·`npm start`를 기준으로 빌드·실행 방식을 제공함. |

---

## 3. 프론트엔드

### Vite
| 항목 | 내용 |
|------|------|
| **역할** | 프론트엔드 빌드 도구 및 개발 서버. React 앱의 번들링·HMR(빠른 새로고침), 프로덕션용 `dist` 생성. |
| **선정 사유** | 기존 CRA 대비 빌드·개발 서버 속도가 빠르고 설정이 단순함. React 공식 문서에서도 권장하는 방식 중 하나. |

### React
| 항목 | 내용 |
|------|------|
| **역할** | UI 라이브러리. 로그인·회원가입·메인(재고·매입·매출·수금·상품 마스터) 등 화면을 컴포넌트 단위로 구성하고 상태 관리. |
| **선정 사유** | 사용처가 많고 자료가 풍부함. 컴포넌트 재사용과 생태계(라우터, 상태 관리 등)가 잘 갖춰져 있어 유지보수·확장에 유리함. |

---

## 4. 백엔드

### Express
| 항목 | 내용 |
|------|------|
| **역할** | Node.js 기반 웹·API 서버 프레임워크. 로그인·회원가입·재고·매입·매출·수금·상품 마스터 등 REST API 제공 및 정적 라우트(/, /health) 처리. |
| **선정 사유** | Node 생태계에서 가장 널리 쓰이는 서버 프레임워크. 가벼우면서 미들웨어·라우팅 구조가 단순해 학습·배포가 쉽고, Railway 등에서 기본 지원됨. |

### pg (node-pg)
| 항목 | 내용 |
|------|------|
| **역할** | Node에서 PostgreSQL에 접속하는 클라이언트. Express에서 SQL 쿼리 실행·연결 풀 관리. |
| **선정 사유** | PostgreSQL 공식 권장 클라이언트에 가깝고, 풀링·프로미스 기반 API 지원. 문서와 사용 사례가 많아 연동·트러블슈팅이 수월함. |

### dotenv
| 항목 | 내용 |
|------|------|
| **역할** | `.env` 파일을 읽어 `process.env`에 주입. 로컬에서는 DB URL·비밀번호 등 비공개 설정을 코드 밖에서 관리. |
| **선정 사유** | 설정과 코드 분리, 보안(민감 정보 미커밋)에 필수. Railway 등은 Variables로 환경 변수를 주입하므로 프로덕션에서는 dotenv 없이도 동작하고, 로컬만 `.env`로 통일 가능. |

## 5. 데이터베이스

### PostgreSQL
| 항목 | 내용 |
|------|------|
| **역할** | 관계형 DB. 회원·거래처·상품·매입·매출·수금·재고 등 영구 저장. 로컬 `sales_tool` DB와 Railway 상의 Postgres 서비스에서 사용. |
| **선정 사유** | 오픈소스·무료. 트랜잭션·무결성·JSON 등 기능이 충실하고, Railway·Render 등에서 관리형 Postgres를 쉽게 붙일 수 있어 개발·배포 모두에 적합함. |

---

## 6. 배포·인프라

### Railway
| 항목 | 내용 |
|------|------|
| **역할** | 백엔드(Node/Express)와 PostgreSQL을 클라우드에서 실행·관리. GitHub 푸시 시 자동 빌드·배포, 공개 URL(도메인) 제공, DB와 앱을 같은 프로젝트 내에서 연결. |
| **선정 사유** | 백엔드와 DB를 한 플랫폼에서 구성할 수 있어 설정이 단순함. 무료 체험/크레딧으로 소규모·학습용으로 시작하기 좋고, Variables·내부 네트워크로 DB URL을 안전하게 주입 가능. |

### Vercel (웹 배포 예정)
| 항목 | 내용 |
|------|------|
| **역할** | React 빌드 결과물(정적 파일) 호스팅. 사용자가 접속하는 웹 주소 제공, 환경 변수로 `VITE_API_URL`(Railway 백엔드 URL) 설정. |
| **선정 사유** | 프론트엔드 배포에 특화되어 있고, GitHub 연동·무료 Hobby 플랜으로 개인·소규모 프로젝트에 적합. `VITE_*` 환경 변수와 Vite 빌드가 잘 맞음. |

---

## 7. 구성 요약

| 구분 | 서비스 | 역할 한 줄 |
|------|--------|-------------|
| 버전 관리 | Git, GitHub | 로컬·원격 코드 이력 및 배포 연동 |
| 런타임·패키지 | Node.js, npm | 서버·빌드 실행 및 의존성 관리 |
| 프론트 | Vite, React | 빌드·개발 서버 및 UI 구현 |
| 백엔드 | Express, pg, dotenv | API 서버, DB 접속, 로컬 환경 설정 |
| DB | PostgreSQL | 영구 데이터 저장 |
| 배포 | Railway, (Vercel) | 백엔드+DB 호스팅, (웹 호스팅) |

이 문서는 프로젝트에서 사용 중인 서비스를 기준으로 작성되었으며, 추후 서비스가 추가·변경되면 함께 갱신합니다.
