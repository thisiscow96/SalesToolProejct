# 시스템별 사용용도

판매툴 프로젝트에서 사용 중인 각 시스템(서비스·기술)의 **사용용도**를 한눈에 정리한 문서입니다.  
역할·선정 사유는 [서비스별 역할 및 선정 사유](서비스별-역할-및-선정사유.md)를 참고하세요.

---

## 요약 표

| 시스템 | 사용용도 |
|--------|----------|
| **Git** | 로컬 코드 버전 관리, 커밋·브랜치 이력 보관 |
| **GitHub** | 원격 저장소, 백업·협업, Render/Vercel 배포 연동 |
| **Node.js** | 백엔드(Express)·프론트(Vite) 실행 런타임 |
| **npm** | 패키지 설치·의존성 관리, `npm start` / `npm run build` 실행 |
| **Vite** | 프론트 빌드·개발 서버, 프로덕션 dist 생성 |
| **React** | 로그인·회원가입·재고·매입·매출·수금·상품 마스터 등 UI 구현 |
| **Express** | REST API 서버, /health·/api/* 라우트 처리 |
| **pg (node-pg)** | PostgreSQL 접속·쿼리·연결 풀 |
| **dotenv** | 로컬 `.env` 로 DB URL·비밀번호 등 환경 변수 로드 |
| **PostgreSQL** | 회원·거래처·상품·매입·매출·수금·재고 등 영구 저장 |
| **Resend** | 회원가입 인증 메일 발송 (배포 환경, API 기반) |
| **nodemailer + Gmail SMTP** | 로컬 개발 시 인증 메일 발송 (Resend 미사용 시) |
| **Render** | 백엔드(Express)·PostgreSQL 호스팅, GitHub 푸시 시 자동 배포 |
| **Vercel** | 프론트(React 빌드 결과물) 호스팅, 사용자 접속 URL·/api rewrite |

---

## 구분별 사용용도

### 개발·버전 관리
- **Git** — 로컬 버전 관리  
- **GitHub** — 원격 저장소·배포 연동  

### 런타임·패키지
- **Node.js** — 서버·빌드 실행  
- **npm** — 의존성·스크립트 실행  

### 프론트엔드
- **Vite** — 빌드·개발 서버  
- **React** — 화면·상태·라우팅  

### 백엔드
- **Express** — API·라우팅  
- **pg** — DB 쿼리  
- **dotenv** — 로컬 환경 변수  

### 데이터베이스
- **PostgreSQL** — 영구 데이터 저장  

### 이메일
- **Resend** — 배포 환경 인증 메일  
- **nodemailer/Gmail SMTP** — 로컬 인증 메일  

### 배포·인프라
- **Render** — 백엔드·DB 호스팅  
- **Vercel** — 프론트 호스팅·공개 URL  

이 문서는 [서비스별 역할 및 선정 사유](서비스별-역할-및-선정사유.md)와 함께 갱신합니다.
