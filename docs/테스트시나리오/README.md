# 테스트 시나리오

API·화면 검증을 위한 **시나리오(케이스) 정의**를 둡니다.

| 문서 | 설명 |
|------|------|
| [API 통합 시나리오](API-통합-시나리오.md) | 백엔드 API 호출 순서·**기대 결과·성공 여부·에러 사유·시행일자** 컬럼 포함 목록 + 상세 시행 기록 |
| [API 통합 시나리오 — 실행 통합 대장](API-통합-시나리오-실행-통합대장.md) | `npm run scenario` **실행 이력 표** — 진행·통과 %, 링크(상세/결과표), 수동 조치·원인(`통합대장-수동필드.json`) |
| [로컬 스모크 실행 (입력 정리)](로컬-스모크-실행.md) | PowerShell에 **무엇을 입력할지** — `npm start`, `npm run smoke`, 포트 3001, `Cannot POST` 대응 |
| [시나리오 실행 이력](시나리오-실행이력/README.md) | `npm run scenario` 결과 자동 저장 — 단계별 진행·실패 ID·치명 오류·`history.jsonl` |

**목록 표 컬럼:** 각 시나리오마다 기대 결과(요약), 성공 여부(`○`/`×`/미시행 등), 실패 시 에러 사유, 시행일자(`YYYY-MM-DD`)를 수동으로 갱신합니다. 상세(§2)에도 시나리오별 **시행 기록** 블록이 있습니다.

**자동 실행:** 저장소 `server`에서 서버 기동 후 `npm run scenario` → S-001~S-021을 `admin001`로 점검합니다. ([로컬 스모크 실행](로컬-스모크-실행.md) §3-1)

이후 자동화 시 이 문서의 시나리오 ID를 테스트 코드(`describe`/`it`)와 1:1로 맞추면 추적이 쉽습니다.
