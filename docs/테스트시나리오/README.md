# 테스트 시나리오

API·화면 검증을 위한 **시나리오(케이스) 정의**를 둡니다.

| 문서 | 설명 |
|------|------|
| [API 통합 시나리오](API-통합-시나리오.md) | 백엔드 API 호출 순서·**기대 결과·성공 여부·에러 사유·시행일자** 컬럼 포함 목록 + 상세 시행 기록 |

**목록 표 컬럼:** 각 시나리오마다 기대 결과(요약), 성공 여부(`○`/`×`/미시행 등), 실패 시 에러 사유, 시행일자(`YYYY-MM-DD`)를 수동으로 갱신합니다. 상세(§2)에도 시나리오별 **시행 기록** 블록이 있습니다.

이후 자동화 시 이 문서의 시나리오 ID를 테스트 코드(`describe`/`it`)와 1:1로 맞추면 추적이 쉽습니다.
