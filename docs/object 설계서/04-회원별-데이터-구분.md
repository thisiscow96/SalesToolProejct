# 회원별 데이터 구분 (user_id)

상품 정보는 **모든 회원이 공통으로 보며**, 금액·거래 데이터는 **해당 데이터를 생성한 회원만** 볼 수 있도록 하기 위한 정책입니다.

---

## 원칙

- **상품 마스터(products)**: `user_id` 없음 → **전체 회원 공통 조회**
- **금액·거래 관련**: `user_id` 필수 → **그 회원이 생성한 데이터만 조회/수정**

---

## user_id가 추가된 객체

| 테이블 | 설명 |
|--------|------|
| **partners** | 거래처는 회원별로 소유. 내 구매처/판매처만 조회 |
| **product_daily_prices** | 일별 단가(금액)는 회원별. 유일 제약 (user_id, product_id, price_date) |
| **purchases** | 구매 건은 생성한 회원만 |
| **sales** | 판매 건은 생성한 회원만 |
| **payments** | 수금 건은 해당 회원만 |
| **disposals** | 폐기 건은 해당 회원만 |
| **inventory** | 재고는 회원별·상품별 1행. 유일 제약 (user_id, product_id) |
| **product_transfers** | 이동 이력은 해당 회원만 |

---

## user_id가 없는 객체

| 테이블 | 이유 |
|--------|------|
| **products** | 상품 목록(이름, 단위, 카테고리)은 전체 회원이 공통으로 조회 |

---

## API·화면에서 할 일

- **로그인한 회원**의 `user_id`를 세션/토큰에서 꺼내서 사용.
- **조회**: 금액·거래 관련 테이블 조회 시 `WHERE user_id = :current_user_id` 필수.
- **등록/수정**: INSERT/UPDATE 시 `user_id = :current_user_id` 설정.
- **receivables_by_partner** 뷰: `user_id` 컬럼 포함. 조회 시 `WHERE user_id = :current_user_id` 조건으로 사용.
