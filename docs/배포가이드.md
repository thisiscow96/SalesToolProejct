# 판매툴 배포 가이드 (Railway + Vercel)

- **백엔드**: Railway (Node + PostgreSQL)
- **프론트엔드**: Vercel (React/Vite)

---

## 1. 백엔드 배포 (Railway)

### 1-1. Railway 가입 및 프로젝트 생성

1. https://railway.app 접속 후 로그인( GitHub 권장 ).
2. **New Project** → **Deploy from GitHub repo** 선택 후 이 저장소 연결.
3. 저장소에서 **server** 폴더만 배포하려면:
   - **Settings** → **Root Directory**를 `server`로 설정.

### 1-2. PostgreSQL 추가

1. 같은 프로젝트에서 **New** → **Database** → **PostgreSQL** 추가.
2. PostgreSQL 서비스 클릭 → **Variables** 탭에서 `DATABASE_URL` 복사.
3. 백엔드(Express) 서비스 **Variables**에 아래 추가:

| 변수명 | 값 |
|--------|-----|
| `DATABASE_URL` | (PostgreSQL에서 복사한 URL) |
| `PORT` | `3000` (Railway가 지정해 줄 수도 있음) |
| `SMTP_HOST` | (선택) smtp.gmail.com 등 |
| `SMTP_PORT` | (선택) 587 |
| `SMTP_USER` | (선택) 이메일 주소 |
| `SMTP_PASS` | (선택) 앱 비밀번호 |
| `SMTP_FROM` | (선택) 발신 이메일 |

### 1-3. 시작 명령 설정

백엔드 서비스 **Settings** → **Deploy**:

- **Start Command**: `npm run migrate && npm start`  
  (배포 시 마이그레이션 자동 실행 후 서버 시작)

### 1-4. 도메인 확인

- **Settings** → **Networking** → **Generate Domain** 클릭.
- 생성된 URL 예: `https://xxx.up.railway.app`  
  → 이 주소를 **프론트엔드 환경 변수**에 사용합니다.

---

## 2. 프론트엔드 배포 (Vercel)

### 2-1. Vercel 프로젝트 생성

1. https://vercel.com 로그인( GitHub 권장 ).
2. **Add New** → **Project** → 이 저장소 Import.
3. **Root Directory**를 `web`으로 설정.
4. **Framework Preset**: Vite 인식되면 그대로 두고 **Deploy**.

### 2-2. API 주소 설정

배포 후 **Project** → **Settings** → **Environment Variables**:

| Name | Value |
|------|--------|
| `VITE_API_URL` | `https://xxx.up.railway.app` (Railway에서 생성한 백엔드 URL, 끝에 `/` 없이) |

저장 후 **Redeploy** 한 번 실행.

---

## 3. 배포 후 확인

1. **백엔드**: `https://[Railway도메인]/health` 접속 → `{"ok":true,"db":"connected"}` 확인.
2. **프론트**: Vercel 배포 URL 접속 → 로그인/회원가입 동작 확인.
3. 로그인 후 재고·매입·매출·수금이 **중매인 번호(agent_no)** 기준으로 조회되는지 확인.

---

## 4. 요약

| 구분 | 서비스 | URL |
|------|--------|-----|
| 백엔드 | Railway | Settings → Networking → Generate Domain |
| 프론트 | Vercel | 배포 후 할당 URL |
| 연동 | Vercel 환경변수 | `VITE_API_URL` = Railway 백엔드 URL |

Git에 push하면 Railway·Vercel 모두 자동 재배포됩니다 (연결된 경우).
