# 판매툴 배포 가이드 (Railway + Vercel)

- **백엔드**: Railway (Node + PostgreSQL) — *Gmail SMTP는 Railway에서 ETIMEDOUT 나면 1-B(Render) 사용*
- **프론트엔드**: Vercel (React/Vite)

---

## 1. 백엔드 배포 (Railway)

### 1-1. Railway 가입 및 프로젝트 생성

1. https://railway.app 접속 후 로그인( GitHub 권장 ).
2. **New Project** → **Deploy from GitHub repo** 선택 후 이 저장소 연결.
3. 저장소에서 **server** 폴더만 배포하려면:
   - **Settings** → **Root Directory**를 `server`로 설정.

### 1-2. PostgreSQL 추가

1. 같은 프로젝트에서 **New** → **Database** → **PostgreSQL** 추가.
2. PostgreSQL 서비스 클릭 → **Variables** 탭에서 `DATABASE_URL` 복사.
3. 백엔드(Express) 서비스 **Variables**에 아래 추가:

| 변수명 | 값 |
|--------|-----|
| `DATABASE_URL` | (PostgreSQL에서 복사한 URL) |
| `PORT` | `3000` (Railway가 지정해 줄 수도 있음) |
| `SMTP_HOST` | smtp.gmail.com |
| `SMTP_PORT` | 587 또는 465 (465 사용 시 SSL) |
| `SMTP_USER` | Gmail 주소 |
| `SMTP_PASS` | Gmail 앱 비밀번호 |
| `SMTP_FROM` | 발신 이메일 |

### 1-3. 시작 명령 설정 (필수 — 테이블 생성)

백엔드 서비스 **Settings** → **Deploy**:

- **Root Directory**: `server` (반드시 설정)
- **Start Command**: `npm run migrate && npm start`  
  (이걸 설정해야 DB에 `user` 등 테이블이 생성됩니다. 설정 안 하면 500 / relation "user" does not exist 발생)

### 1-4. 도메인 확인

- **Settings** → **Networking** → **Generate Domain** 클릭.
- 생성된 URL 예: `https://xxx.up.railway.app`  
  → 이 주소를 **프론트엔드 환경 변수**에 사용합니다.

## 1-B. 백엔드 배포 대안 — Render (Gmail SMTP 사용 시)

Railway는 아웃바운드 SMTP가 막혀 Gmail 발송 시 **ETIMEDOUT** 이 납니다. **Gmail SMTP를 그대로 쓰려면** Node 백엔드를 **Render**로 옮기면 됩니다.

1. **https://render.com** 가입( GitHub 연동 권장 ).
2. **Dashboard** → **New +** → **Web Service**.
3. **Connect a repository** → 이 GitHub 저장소 연결.
4. 설정:
   - **Name**: 예) `sales-tool-api`
   - **Root Directory**: `server`
   - **Runtime**: Node
   - **Build Command**: `npm install`
   - **Start Command**: `npm run migrate && npm start`
5. **Advanced** → **Add Environment Variable** 에서 추가:

   | Key | Value |
   |-----|--------|
   | `DATABASE_URL` | (아래 6번에서 만든 Postgres URL) |
   | `SMTP_HOST` | smtp.gmail.com |
   | `SMTP_PORT` | 587 또는 465 |
   | `SMTP_USER` | Gmail 주소 |
   | `SMTP_PASS` | Gmail 앱 비밀번호 |
   | `SMTP_FROM` | 발신 이메일 |

6. **PostgreSQL 생성 (같은 Render 대시보드에서)**  
   - **Dashboard** (왼쪽 상단 로고 클릭) → **New +** → **PostgreSQL** 선택.  
   - Name 예: `sales-tool-db` → **Create Database** 클릭.  
   - 생성된 DB 서비스 클릭 → **Info** 탭 → **Internal Database URL** 항목 옆 **복사** 버튼으로 URL 복사.  
   - 이 값이 `DATABASE_URL` 이다. 형식 예: `postgresql://사용자:비밀번호@호스트/DB이름?sslmode=require` (실제 값은 Render가 만들어 주므로, 여기서 복사한 걸 그대로 쓴다.)  
   - Web Service 쪽 **Environment** 에 `DATABASE_URL` = (방금 복사한 URL) 로 넣기.  
   - 반드시 **Internal Database URL** 을 쓸 것. (External 이 아닌 Internal. Web Service와 DB가 같은 Render 안에서 통신해야 함.)
7. **Create Web Service** 후 배포 완료되면 **URL** 확인 (예: `https://salestoolproejct.onrender.com`).

**Postgres 서비스 "연결"은 어디서 하나요?**  
- **Web Service** 생성/수정 화면에서 아래로 내리면 **Environment** 섹션이 나옵니다.  
- 그 안에 **"Add from" / "Link resource" / "Connect database"** 같은 버튼이 있으면, 거기서 **기존 Postgres 서비스**를 고르면 `DATABASE_URL` 가 자동으로 붙습니다.  
- 그런 항목이 없으면, Postgres 서비스 → **Info** → **Internal Database URL** 을 복사해서, Web Service **Environment** 에서 **Add Environment Variable** → Key `DATABASE_URL`, Value에 붙여넣기 하면 됩니다.

**"DB 연결 실패" 나올 때:**  
- **Internal Database URL** 썼는지, Web Service **Environment** 에 `DATABASE_URL` 가 들어가 있는지 확인.
8. **Vercel** 쪽 연동:
   - **vercel.json** 의 `/api` rewrite 목적지를 이 Render URL로 변경하거나,
   - 또는 Vercel **Environment Variables** 에 `VITE_API_URL` = `https://salestoolproejct.onrender.com` (Render URL) 로 설정 후 Redeploy.

이렇게 하면 백엔드는 **Node(Render)** 에 올라가고, Gmail SMTP로 인증 메일 발송이 됩니다.

## 2. 프론트엔드 배포 (Vercel)

### 2-1. Vercel 프로젝트 생성

1. https://vercel.com 로그인( GitHub 권장 ).
2. **Add New** → **Project** → 이 저장소 Import.
3. **Root Directory**를 `web`으로 설정.
4. **Framework Preset**: Vite 인식되면 그대로 두고 **Deploy**.

### 2-2. API 주소 설정

배포 후 **Project** → **Settings** → **Environment Variables**:

| Name | Value |
|------|--------|
| `VITE_API_URL` | `https://xxx.up.railway.app` (Railway에서 생성한 백엔드 URL, 끝에 `/` 없이) |

저장 후 **Redeploy** 한 번 실행.

## 3. 배포 후 확인

1. **백엔드**: `https://[Railway도메인]/health` 접속 → `{"ok":true,"db":"connected"}` 확인.
2. **프론트**: Vercel 배포 URL 접속 → 로그인/회원가입 동작 확인.
3. 로그인 후 재고·매입·매출·수금이 **중매인 번호(agent_no)** 기준으로 조회되는지 확인.

## 5. 문제 해결

### "relation \"public.user\" does not exist" / 500

Railway DB에 테이블이 없는 상태입니다.

1. Railway → **백엔드 서비스** 클릭 → **Settings**
2. **Root Directory**: `server` 로 설정
3. **Start Command**: `npm run migrate && npm start` 로 설정
4. **Variables**에 `DATABASE_URL` 이 PostgreSQL 서비스에서 복사한 값인지 확인
5. **Redeploy** 실행 (Deployments → ⋯ → Redeploy)

재배포 시 시작 단계에서 마이그레이션이 돌며 `user`, `account` 등 테이블이 생성됩니다.

### 이메일이 안 갈 때 (Gmail SMTP)

- **ETIMEDOUT / Connection timeout** → Railway는 아웃바운드 SMTP를 막아서 Gmail 발송이 안 됨. **백엔드를 Render로 옮기면** Gmail SMTP 사용 가능 (위 **1-B** 참고).
- Railway 쓸 때: Variables에 SMTP 전부 넣었는지, `SMTP_PORT=465` 시도해 보기. 그래도 ETIMEDOUT 이면 Render로 이전하는 수밖에 없음.
