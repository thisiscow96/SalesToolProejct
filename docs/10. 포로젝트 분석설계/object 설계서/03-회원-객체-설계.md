# 회원(Members) 객체 설계

회원가입, 이메일 인증, 개인정보약관 동의를 위한 객체 설계입니다.

---

## 요구 사항 정리

- **회원가입 시**: 이름, 휴대폰번호, 이메일, 아이디, 비밀번호
- **이메일 인증**: 가입 전/후 이메일 소유 확인
- **개인정보약관 동의**: 동의 시점 기록 필수

---

## 1. 회원 (users)

| 속성 | 타입 | 설명 |
|------|------|------|
| id | serial | PK |
| name | string | 이름 |
| phone | string | 휴대폰번호 |
| email | string | 이메일 (유일) |
| login_id | string | 로그인 아이디 (유일) |
| password_hash | string | 비밀번호 해시 (bcrypt 등, 원문 저장 금지) |
| email_verified_at | timestamptz (nullable) | 이메일 인증 완료 시각. null이면 미인증 |
| terms_agreed_at | timestamptz | 개인정보약관 동의 시각 (가입 시 필수) |
| status | enum/string | 계정 상태: active, suspended, withdrawn |
| created_at | timestamptz | 가입일시 |
| updated_at | timestamptz | 수정일시 |

- **유일 제약**: email, login_id 각각 UNIQUE.
- **비밀번호**: 반드시 해시만 저장. 화면/API에서는 원문 비밀번호를 받아 서버에서 해시 후 저장.

---

## 2. 이메일 인증 (email_verifications)

가입 시 또는 재인증 시 발송한 **인증 코드/토큰**과 만료·완료 시각을 관리.

| 속성 | 타입 | 설명 |
|------|------|------|
| id | serial | PK |
| email | string | 인증 대상 이메일 |
| user_id | FK → users (nullable) | 회원 존재 시 연결 (재인증 등) |
| code | string | 인증 코드 (6자리 숫자 또는 링크용 토큰) |
| expires_at | timestamptz | 만료 시각 |
| verified_at | timestamptz (nullable) | 인증 완료 시각. null이면 미완료 |
| created_at | timestamptz | 발송/생성 시각 |

- **흐름**: 1) 사용자가 이메일 입력 → 코드 발송 → 이 테이블에 1건 등록. 2) 사용자가 코드 입력 → 일치·만료 검사 후 `verified_at` 설정. 3) 가입 완료 시 해당 users 행에 `email_verified_at` 설정.

---

## 3. 약관 동의 이력 (terms_agreements) — 선택

개인정보약관 등 **약관별·시점별 동의 이력**을 남기려면 사용.

| 속성 | 타입 | 설명 |
|------|------|------|
| id | serial | PK |
| user_id | FK → users | 회원 |
| terms_type | string | 약관 구분 (예: privacy, service) |
| agreed_at | timestamptz | 동의 시각 |
| version | string | 약관 버전 (선택) |
| ip_address | string | 동의 시 IP (선택) |

- **최소 구현**: users.terms_agreed_at 만으로 “가입 시 개인정보약관 동의” 처리 가능.
- **확장**: 약관이 여러 종류이거나, 동의 이력을 남겨야 하면 이 테이블 사용.

---

## 관계 요약

```
users (회원)
  ↑
  |
email_verifications (user_id nullable, email 필수)
terms_agreements (user_id) — 선택
```

- **가입 플로우**: 이메일 인증(코드 발송 → 검증) → 회원 정보 + 비밀번호 해시 + terms_agreed_at 저장 → users.email_verified_at 설정.

---

## 화면·API 고려

- **회원가입 화면**: 이름, 휴대폰, 이메일, 아이디, 비밀번호 입력 → (선택) 이메일 인증 요청 → 인증 코드 입력 → 개인정보약관 동의 체크(필수) → 가입 API 호출.
- **이메일 인증**: “인증 메일 보내기” → email_verifications 에 코드·만료 시각 저장 + 메일 발송. “코드 확인” → code·email·만료 검사 후 verified_at 갱신.
- **비밀번호**: API에서만 해시 후 저장. 조회 시에는 절대 반환하지 않음.
