# 외부에서 접속 가능한 URL 만들기

## 1) 로컬을 잠깐 공개 (테스트/공유용)

PC에서 서버·웹을 켜 둔 상태로 **임시 공개 URL**을 쓰는 방법입니다.  
PC를 끄거나 ngrok을 끄면 URL은 더 이상 동작하지 않습니다.

### ngrok 사용

1. https://ngrok.com 가입 후 ngrok 다운로드(또는 `choco install ngrok`).
2. **서버**와 **웹**을 로컬에서 실행해 둡니다.
   - 서버: `cd server` → `npm run dev` (포트 3000)
   - 웹: `cd web` → `npm run dev` (포트 5173)
3. 터미널에서:
   ```bash
   ngrok http 5173
   ```
   - 화면에 나오는 **https://xxxx.ngrok.io** 주소가 **웹(React)** 공개 URL입니다.
   - 웹이 백엔드(3000)를 쓰므로, Vite 프록시(`/api` → 3000)가 **같은 PC**에서만 동작합니다.  
     **다른 기기**에서 ngrok URL로 접속하면 API 요청이 “같은 PC의 3000”으로 가지 않으므로, **백엔드도 함께 공개**해야 합니다.
4. **백엔드도 공개**하려면 터미널 하나 더 열어서:
   ```bash
   ngrok http 3000
   ```
   - 나온 URL(예: https://yyyy.ngrok.io)이 **API 서버 주소**입니다.
   - 웹에서 API를 이 주소로 보내려면, **빌드 시** 또는 **환경 변수**로 `VITE_API_URL=https://yyyy.ngrok.io` 처럼 설정하고, `web`에서 `fetch(import.meta.env.VITE_API_URL + '/api/...')` 형태로 바꿔야 합니다. (현재는 `/api`로 상대 경로 쓰면 “웹과 같은 호스트”로만 요청 감.)

**정리**: ngrok은 “로컬을 잠깐 외부에 열 때” 쓰고, **항상 켜 두고 쓰는 사이트**는 아래처럼 **배포**하는 게 맞습니다.

---

## 2) 실제 배포 (항상 켜 둔 URL)

서버와 웹을 **호스팅 서비스**에 올리면, 외부에서 **고정 URL**로 접속할 수 있습니다.

### 흐름 요약

| 구분 | 할 일 | 추천 서비스 예시 |
|------|--------|------------------|
| DB | PostgreSQL 올리기 | Railway, Render, Supabase, Neon |
| 백엔드 | Node(Express) 올리기, DB 연결 | Railway, Render |
| 프론트 | React 빌드 후 정적 파일 올리기 | Vercel, Netlify, 또는 Railway |

- **한 곳에서 전부**: **Railway** 에서 프로젝트 생성 → GitHub 연결 → `server` 폴더를 서비스로 추가, PostgreSQL 추가, `web`은 빌드 후 정적 서버로 추가하거나 별도 서비스로 배포.
- **나누기**: 백엔드+DB는 **Railway** 또는 **Render**, 웹은 **Vercel** 또는 **Netlify**.

### A. Railway로 백엔드 + DB — 단계별 (어디서 뭘 누르는지)

**1단계: 프로젝트 만들기**

- https://railway.app 접속 → 로그인.
- **New Project** 버튼 클릭.

**2단계: PostgreSQL 추가 (DB)**

- "Add a plugin" 또는 **+ New** 클릭.
- 목록에서 **PostgreSQL** 선택 → 추가.
- 추가되면 화면에 **PostgreSQL 서비스(카드)** 가 생김.  
  → 나중에 백엔드 서비스에서 이 DB의 `DATABASE_URL`을 쓰게 됨.

**3단계: GitHub 레포로 서비스 추가 (백엔드)**

- 다시 **+ New** (또는 "Add a service") 클릭.
- **GitHub Repo** 선택.
- GitHub 권한 허용 후 **저장소 선택** → `thisiscow96/SalesToolProejct` 선택.
- 연결하면 **해당 레포를 쓰는 서비스(카드)** 가 하나 생김.  
  → 이 카드가 "우리 백엔드(Express 서버)"입니다.

**4단계: 이 서비스가 `server` 폴더만 쓰도록 설정**

- 방금 생긴 **백엔드 서비스 카드**를 **클릭**해서 들어갑니다.
- 왼쪽 또는 위 메뉴에서 **Settings** 탭 클릭.
- **Settings** 안에서 아래 항목을 찾아 설정합니다.

| 항목 | 넣을 값 | 설명 |
|------|--------|------|
| **Root Directory** | `server` | 레포 루트가 아니라 `server` 폴더만 빌드/실행하게 함. |
| **Build Command** | `npm install` | 배포 시 실행할 빌드 명령. (필요 없으면 비워도 됨) |
| **Start Command** | `npm start` | 서버를 켜는 명령. (`node index.js` 와 동일) |

- **Root Directory** 가 없으면 **Custom Build** / **Configure** 같은 섹션을 열어보면 나올 수 있음.  
  값은 반드시 `server` (따옴표 없이).

**5단계: DB 연결 (Variables)**

- 같은 백엔드 서비스에서 **Variables** 탭 클릭.
- PostgreSQL을 아까 추가했다면 **Add Variable** 또는 **Connect to PostgreSQL** 같은 버튼이 있음.
- **Connect to PostgreSQL** 또는 **Reference** 를 선택하면 Railway가 `DATABASE_URL` 을 자동으로 넣어 줌.  
  → 이렇게 하면 백엔드가 배포 시 그 DB에 연결됨.
- `PORT` 는 Railway가 알아서 넣어 주는 경우가 많음. 없다면 **Add Variable** → 이름 `PORT`, 값 `3000` 추가.

**6단계: 공개 URL 만들기 (도메인 생성)**

- 같은 백엔드 서비스 안에서 **Settings** 또는 **Networking** 탭으로 이동.
- **Generate Domain** 또는 **Public Networking** → **Generate Domain** 버튼 클릭.
- 생성되면 `https://xxxxx.railway.app` 같은 주소가 나옵니다.  
  → 이 주소가 **백엔드(API) 주소**입니다. (예: `https://sales-tool-server.railway.app`)

**7단계: 배포 확인**

- 상단 **Deployments** 탭에서 배포 로그 확인.  
  빌드/시작이 성공하면 **Success** 또는 **Active** 로 보임.
- 브라우저에서 `https://생성한주소.railway.app` 접속 →  
  `{"ok":true,"message":"Sales tool backend is running"}` 비슷한 JSON이 보이면 정상.

---

### Generate Domain 다음에 할 일 (3가지)

**① 백엔드 URL 확인**

- 브라우저에서 `https://본인도메인.railway.app` 접속.
- `{"ok":true,"message":"Sales tool backend is running"}` 나오면 OK.
- `https://본인도메인.railway.app/health` 도 열어보기.  
  `db: "connected"` 면 DB 연결까지 된 것.

**② Railway PostgreSQL에 스키마 적용**

- 로그인·회원가입을 쓰려면 **테이블**이 있어야 함.
- Railway 대시보드에서 **Postgres-1sVd**(연결된 DB) 카드 클릭 → **Data** 또는 **Connect** 탭에서 **Connection URL** 복사.  
  (또는 SalesToolProejct 서비스 **Variables** 에서 `DATABASE_URL` 값 복사.)
- PC에서 **pgAdmin** 또는 **psql**로 그 URL에 접속해 **새 DB**는 안 만들고, 그 URL에 포함된 DB에 대해 **schema-full.sql** 전체 실행.  
  (Railway PostgreSQL은 이미 DB 하나 만들어져 있으므로, 그 DB에 스키마만 실행하면 됨.)
- 요약: Railway가 준 연결 문자열로 접속 → `schema-full.sql` 내용 붙여넣기 → 실행.

**③ 웹(React) 배포 — Vercel**

- https://vercel.com 가입 후 **Add New** → **Project** → GitHub에서 **SalesToolProejct** 레포 선택.
- **Root Directory** → **Edit** → `web` 입력.
- **Build Command**: `npm run build`  
  **Output Directory**: `dist`
- **Environment Variables** → **Add**  
  - Name: `VITE_API_URL`  
  - Value: `https://본인도메인.railway.app` (Railway에서 만든 주소, 끝에 `/` 없이)
- **Deploy** 클릭.
- 배포 끝나면 **Visit** 또는 나오는 URL(예: `https://salestoolproejct.vercel.app`)이 **사이트 주소**.  
  이 주소로 접속하면 로그인·회원가입·메인 화면이 Railway API를 쓰게 됨.

---

**요약**: GitHub 레포 연결로 **서비스(카드)** 가 생긴 뒤 → 그 **카드 클릭** → **Settings** 에서 Root Directory `server`, Start Command `npm start` → **Variables** 에서 PostgreSQL 연결(`DATABASE_URL`) → **Networking** 에서 **Generate Domain** 으로 URL 생성.

### B. 웹(React) 배포 (Vercel 예시)

1. https://vercel.com 가입 → **Add New** → **Project** → 같은 GitHub 레포 선택.
2. **Root Directory**를 `web` 으로 설정.
3. **Build Command**: `npm run build`  
   **Output Directory**: `dist`
4. **Environment Variables** 에 백엔드 URL 넣기:
   - 이름: `VITE_API_URL`  
   - 값: Railway에서 만든 API 주소 (예: `https://sales-tool-server.railway.app`)
5. 웹 코드에서 API 호출을 환경 변수로 바꿉니다 (아래 참고).
6. 배포 후 나오는 URL(예: `https://sales-tool-web.vercel.app`)이 **사이트 접속 주소**.

### 웹에서 외부 API 주소 쓰기

- 개발: `vite.config.js` 의 proxy로 `/api` → `localhost:3000` 유지.
- 배포: `VITE_API_URL` 이 있으면 그걸 쓰고, 없으면 `/api`(상대 경로) 쓰도록 하면 됩니다.

`web/src/api.js` (또는 API 베이스 쓰는 파일) 상단:

```js
const API_BASE = import.meta.env.VITE_API_URL
  ? import.meta.env.VITE_API_URL + '/api'
  : '/api';
```

이렇게 하면:
- 로컬: `VITE_API_URL` 없음 → `/api` (프록시로 3000)
- Vercel: `VITE_API_URL=https://sales-tool-server.railway.app` → 해당 URL + `/api`

## 한 줄 요약

- **잠깐만 외부에서 접속**: ngrok으로 `http 5173`(웹), 필요 시 `http 3000`(API) 공개하고, 웹은 API URL을 ngrok API 주소로 맞춰 주면 됨.
- **항상 켜 둔 사이트 + URL 배포**: Railway(백엔드+DB) + Vercel(웹) 배포 후, `VITE_API_URL` 로 API 주소 연결하면 외부에서 URL로 접근 가능.
