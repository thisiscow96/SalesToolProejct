## 2. 서버 켜기 (터미널 1)


## 4. `Cannot GET /api/capabilities` / `Cannot POST /api/purchases` / 스모크가 capabilities에서 막힐 때

**의미:** 지금 이 저장소의 `server/index.js`에는 `GET /api/capabilities`가 **반드시** 있습니다. 브라우저에 `Cannot GET`이면 **3000 포트에서 돌아가는 프로그램이 이 파일이 아닙니다** (옛날 복사본, 다른 프로젝트의 Express, 예전에 켠 `node` 등).

**할 일:** 아래 순서로 **3000을 쓰는 node를 끄고**, 반드시 **`판매툴\server`에서** `npm start`를 다시 실행합니다.

### 4-1. 3000 쓰는 프로세스 끄기 (PowerShell)

```powershell
Get-NetTCPConnection -LocalPort 3000 -ErrorAction SilentlyContinue | Select-Object OwningProcess
```

나온 **PID**를 작업 관리자에서 `node` 로 찾아 종료하거나:

```powershell
Stop-Process -Id <PID> -Force
```

### 4-2. 최신 코드 받고 다시 서버

```powershell
cd "C:\Users\gunme\Desktop\건우\프로젝트\3. 판매툴"
git pull
cd server
npm start
```

다시 브라우저에서 **`/api/capabilities`** 확인 후:

```powershell
npm run smoke
```

### 4-3. 3000이 안 비워질 때 → **3001** 로만 띄우기

**터미널 1 (서버):**

```powershell
cd "C:\Users\gunme\Desktop\건우\프로젝트\3. 판매툴\server"
$env:PORT = "3001"
npm start
```

**터미널 2 (스모크):**

```powershell
cd "C:\Users\gunme\Desktop\건우\프로젝트\3. 판매툴\server"
$env:SMOKE_API_BASE = "http://127.0.0.1:3001"
npm run smoke
```

## 5. 입력 한눈에 (순서)

| 순서 | 어디 | 입력 |
|------|------|------|
| 1 | `server` | `npm run migrate` (필요 시) |
| 2 | `server` | `npm run seed-admin` (최초 또는 admin 없을 때) |
| 3 | 터미널 A `server` | `npm start` |
| 4 | 브라우저 | `http://127.0.0.1:3000/api/capabilities` → `tradeFlowPost: true` 확인 |
| 5 | 터미널 B `server` | `npm run smoke` |

3000 충돌 시 → **§4-3** 처럼 `PORT=3001` + `SMOKE_API_BASE=http://127.0.0.1:3001` 로 스모크.
