# 이슈관리 — 2025-03-13

## 이메일 중복 확인 — 처음 입력한 이메일도 "이미 사용 중"

### 무슨 이슈였는지

- DB에 해당 이메일이 없는데도 **"이미 사용 중인 이메일입니다"** 로 표시됨.

### 원인

- **프론트**: 서버 오류(500, 네트워크 오류) 시에도 `data.available === false`이면 무조건 "이미 사용 중"으로 표시.
- **백엔드**: 500/에러 응답에 사용자용 메시지가 없어, 클라이언트가 원인 구분 불가.

### 어떤 해결을 했는지

- `!res.ok` 일 때는 **"확인할 수 없습니다"** 등으로 표시하고, **"이미 사용 중"**은 **200 응답이면서 `available === false`일 때만** 표시하도록 수정.
- 백엔드 500 응답에 `message`, 필요 시 `error`(디버깅용) 필드 포함.
