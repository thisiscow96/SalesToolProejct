# 이슈관리 — 2026-03-15

## 1. 이메일 인증 — Gmail SMTP가 PaaS에서 안 됨 (ETIMEDOUT)

### 무슨 이슈였는지

- 회원가입 시 "인증번호 발송" 클릭 시 **발송 중**에서 멈추거나, **테스트용 인증번호**만 표시되고 실제 메일이 오지 않음.
- 로그/응답: **ETIMEDOUT** (Connection timeout).

### 원인

- **Railway**, **Render** 모두 아웃바운드 **SMTP(587, 465)** 를 막거나 제한함.
- Gmail SMTP(smtp.gmail.com)로 나가는 TCP 연결이 PaaS 방화벽에서 차단되어 타임아웃 발생.
- Render로 옮겨도 SMTP 미허용으로 동일 증상.

### 어떤 해결을 했는지

- **Resend** 등 **HTTPS 기반 이메일 API** 사용. SMTP 포트 없이 API 호출만으로 발송.
- `RESEND_API_KEY` 설정 시 Resend로 발송, 미설정 시 로컬에서는 기존 Gmail SMTP(nodemailer) 사용.

## 2. Vercel 배포 — API 요청 405 / 500

### 무슨 이슈였는지

- 배포된 사이트에서 API 호출 시 **405 Method Not Allowed** 또는 **500 Internal Server Error**.
- 요청 URL이 `https://sales-tool-proejct.vercel.app/api/...` 로 나감.

### 원인

- **405**: `vercel.json` 의 rewrite가 없거나, `/api` 를 백엔드가 아닌 정적 호스트로 보냄.
- **500**: rewrite의 **destination** 이 예전 백엔드 URL로 되어 있거나, 백엔드를 Render로 바꾼 뒤 **vercel.json**·**VITE_API_URL** 미갱신으로 옛 백엔드로 요청 감.

### 어떤 해결을 했는지

- `vercel.json` 에서 `/api/:path*` 의 **destination** 을 실제 사용 중인 백엔드 URL로 설정 (예: `https://salestoolproejct.onrender.com/api/:path*`).
- 백엔드 호스팅을 바꿀 때마다 **Vercel 환경 변수 `VITE_API_URL`** 및 **vercel.json rewrite** 를 함께 갱신 후 Redeploy.
