# 판매툴 배포 가이드 (Render + Vercel)

- **백엔드**: Render (Node + PostgreSQL). 이메일은 **Resend** API 사용 (SMTP 미사용).
- **프론트엔드**: Vercel (React/Vite)

---

## 1. 백엔드 배포 (Render)

### 1-1. Render 가입 및 Web Service 생성

1. https://render.com 가입( GitHub 연동 권장 ).
2. **Dashboard** → **New +** → **Web Service**.
3. **Connect a repository** → 이 GitHub 저장소 연결.
4. 설정:
   - **Name**: 예) `salestoolproejct`
   - **Root Directory**: `server`
   - **Runtime**: Node
   - **Build Command**: `npm install`
   - **Start Command**: `npm run migrate && npm start`

### 1-2. PostgreSQL 추가

1. **Dashboard** (왼쪽 상단 로고) → **New +** → **PostgreSQL**.
2. Name 예: `sales-tool-db` → **Create Database**.
3. 생성된 DB 클릭 → **Info** 탭 → **Internal Database URL** 복사.
4. Web Service **Environment** 에 `DATABASE_URL` = (방금 복사한 URL) 추가.  
   **반드시 Internal Database URL** 사용 (External 아님).  
   또는 Web Service **Environment** 에서 **Connect existing resource** 로 Postgres 연결 시 `DATABASE_URL` 자동 주입.

### 1-3. 환경 변수 (Web Service)

| 변수명 | 값 |
|--------|-----|
| `DATABASE_URL` | Postgres **Internal Database URL** (위에서 복사) |
| `RESEND_API_KEY` | Resend API Keys에서 발급한 키 |
| `RESEND_FROM` | 테스트: `onboarding@resend.dev` / 실서비스: 도메인 인증 후 `noreply@garaksalestool.com` 등 |

- **Start Command**: `npm run migrate && npm start` (미설정 시 DB 테이블 없어 500 발생 가능).

### 1-4. 도메인 확인

- 배포 완료 후 **URL** 확인 (예: `https://salestoolproejct.onrender.com`).  
  → 이 주소를 **프론트엔드 환경 변수** 및 **vercel.json** 에 사용.

---

## 2. 프론트엔드 배포 (Vercel)

### 2-1. Vercel 프로젝트 생성

1. https://vercel.com 로그인( GitHub 권장 ).
2. **Add New** → **Project** → 이 저장소 Import.
3. **Root Directory**를 `web`으로 설정.
4. **Framework Preset**: Vite 인식되면 그대로 **Deploy**.

### 2-2. API 주소 설정

**Project** → **Settings** → **Environment Variables**:

| Name | Value |
|------|--------|
| `VITE_API_URL` | `https://salestoolproejct.onrender.com` (Render 백엔드 URL, 끝에 `/` 없이). 커스텀 도메인 사용 시 `https://garaksalestool.com` |

저장 후 **Redeploy** 실행.

- **vercel.json** 에서 `/api/:path*` 의 **destination** 이 위 Render URL(또는 커스텀 도메인)의 `/api/:path*` 로 되어 있는지 확인.

---

## 3. 배포 후 확인

1. **백엔드**: `https://[Render도메인]/health` 접속 → `{"ok":true,"db":"connected"}` 확인.
2. **프론트**: Vercel 배포 URL(또는 https://garaksalestool.com) 접속 → 로그인/회원가입 동작 확인.
3. 로그인 후 재고·매입·매출·수금이 **중매인 번호(agent_no)** 기준으로 조회되는지 확인.

---

## 4. 요약

| 구분 | 서비스 | URL |
|------|--------|-----|
| 백엔드 | Render | Web Service 할당 URL |
| DB | Render PostgreSQL | Internal URL → `DATABASE_URL` |
| 프론트 | Vercel | 배포 후 할당 URL 또는 garaksalestool.com |
| 연동 | Vercel 환경변수 + vercel.json | `VITE_API_URL`, `/api` rewrite → Render |

Git push 시 Render·Vercel 자동 재배포(연결된 경우).

## 5. 문제 해결

### "relation \"public.user\" does not exist" / 500

DB에 테이블이 없음.

1. Render → Web Service → **Environment** 에 `DATABASE_URL` (Internal URL) 확인.
2. **Start Command**: `npm run migrate && npm start` 확인.
3. **Manual Deploy** 또는 **Redeploy** 실행.

### 이메일이 안 갈 때

- **Resend** 사용: Render **Environment** 에 `RESEND_API_KEY`, `RESEND_FROM` 설정 후 Redeploy.
- 아무 이메일로 보내려면 Resend에 도메인 인증 후 `RESEND_FROM` = `noreply@garaksalestool.com` 등으로 설정.  
  자세한 절차: [Resend 설정 가이드](../아키텍쳐%20구성/도메인/Resend-설정-가이드.md), [환경 변수](../환경%20변수설정/README.md).

### Render — DB 연결 실패 (마이그레이션 실패)

- **Internal Database URL** 사용 여부 확인. **Connect existing resource** 로 Postgres 연결 또는 수동으로 `DATABASE_URL` 입력.
- 마이그레이션 스크립트에서 `render.com` / `sslmode` 시 `ssl: { rejectUnauthorized: false }` 적용되어 있는지 확인.

### Vercel — API 405 / 500

- `vercel.json` 의 `/api/:path*` **destination** 이 현재 백엔드 URL인지 확인.  
  Render 사용 시 `https://salestoolproejct.onrender.com/api/:path*` (또는 커스텀 도메인).
- `VITE_API_URL` 변경 시 Redeploy 필요.
