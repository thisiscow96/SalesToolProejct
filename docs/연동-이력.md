# 연동 이력 (Git · Node · 백엔드 · DB)

판매툴 프로젝트에서 **Git 연동, Node 연동, 백엔드(Express) 및 DB(PostgreSQL) 연동**을 진행하면서 한 작업과 **시행착오·해결 방법**을 정리한 문서입니다.  
이력을 GitHub에 올려 두어 나중에 참고하거나 신규 환경 세팅 시 활용할 수 있습니다.

---

## 1. Git 연동

### 1.1 진행한 작업
- 로컬 프로젝트 폴더에 `git init` 으로 저장소 생성
- GitHub 레포지토리 생성: `https://github.com/thisiscow96/SalesToolProejct`
- `git remote add origin <URL>` 로 원격 연결
- `git branch -M main` 으로 기본 브랜치를 `main` 으로 통일
- `git add .` → `git commit` → `git push -u origin main` 으로 코드 푸시

### 1.2 시행착오 및 해결

| 현상 | 원인 | 해결 |
|------|------|------|
| `git` / `npm` 을 찾을 수 없다 (PowerShell) | Git·Node 설치 후 PATH가 해당 터미널에 반영되지 않음 | **새 PowerShell/CMD**를 열거나, Cursor를 재시작한 뒤 다시 실행. 또는 **CMD(명령 프롬프트)** 에서 실행 |
| `npm` 실행 시 "스크립트를 실행할 수 없음" (PowerShell) | ExecutionPolicy로 인해 `npm.ps1` 실행 차단 | **CMD에서 작업**하거나, 관리자 PowerShell에서 `Set-ExecutionPolicy -Scope CurrentUser RemoteSigned` 실행 |
| `failed to push some refs` | 원격에 없는 커밋이 있거나, 인증 실패 등 | `git pull origin main --allow-unrelated-histories` 후 `git push` 또는, 인증(토큰) 확인 |
| `src refspec main does not match any` | 로컬 브랜치 이름이 `master` 이거나, 커밋이 없음 | `git branch` 로 확인 후 `git branch -M main` 또는 최소 1회 `git commit` 후 푸시 |
| GitHub 로그인/비밀번호 요청, "매칭 안 됨" | GitHub는 비밀번호 대신 **Personal Access Token** 사용 | GitHub → Settings → Developer settings → Personal access tokens 에서 **repo** 권한 토큰 발급 후, 비밀번호 칸에 **토큰** 입력. Windows 자격 증명에 예전 비밀번호가 있으면 **자격 증명 관리자**에서 `git:https://github.com` 제거 후 다시 푸시 |
| `No configured push destination` | `origin` 미설정 또는 upstream 미설정 | `git remote add origin https://github.com/thisiscow96/SalesToolProejct.git` 후 `git push -u origin main` |
| `couldn't find remote ref main` (pull 시) | 원격 레포가 비어 있어 `main` 브랜치가 없음 | pull 없이 **push만** 수행. `git push -u origin main` 으로 원격에 `main` 생성 |

### 1.3 주의 사항
- **`.env`** 파일은 비밀번호·연결 정보가 있으므로 **GitHub에 올리지 않음**. `.gitignore` 에 이미 포함됨.
- 푸시 전 `git status` 로 `.env` 가 포함되지 않았는지 확인하는 것이 좋음.

## 2. Node 연동

### 2.1 진행한 작업
- Node.js LTS 버전 설치 (https://nodejs.org)
- 프로젝트 루트에서 `npm create vite@latest web -- --template react` 로 웹(React) 프로젝트 생성
- `web/` 폴더에서 `npm install` → `npm run dev` 로 개발 서버 실행 (http://localhost:5173)
- `server/` 폴더에서 `npm install` 로 Express·pg·dotenv 등 설치

### 2.2 시행착오 및 해결

| 현상 | 원인 | 해결 |
|------|------|------|
| `npm` 용어가 인식되지 않음 | Node.js 미설치 또는 PATH 미반영 | Node.js 설치 후 **새 터미널**에서 `node -v`, `npm -v` 확인 |
| PowerShell에서 npm 실행 시 보안 오류 | ExecutionPolicy | CMD에서 실행하거나, `Set-ExecutionPolicy -Scope CurrentUser RemoteSigned` |
| `Cannot find module 'dotenv'` | `server/` 에서 `npm install` 미실행 | `cd server` 후 `npm install` 한 번 실행 |

## 3. 백엔드(Express) · DB(PostgreSQL) 연동

### 3.1 진행한 작업
- PostgreSQL 설치 (서버 + pgAdmin, 필요 시 Command Line Tools)
- DB `sales_tool` 생성 (pgAdmin 또는 `CREATE DATABASE sales_tool;`)
- `server/` 에 Express 서버 구성 (`index.js`), `pg` 로 PostgreSQL 연결
- 환경 변수로 연결 정보 관리: `.env` (DATABASE_URL 또는 PG_HOST, PG_USER, PG_PASSWORD, PG_DATABASE)
- `GET /`, `GET /health` 로 서버·DB 연결 상태 확인

### 3.2 시행착오 및 해결

| 현상 | 원인 | 해결 |
|------|------|------|
| `SASL: client password must be a string` | 비밀번호가 문자열로 전달되지 않음 (빈 값 또는 URL 파싱 문제) | `.env` 에서 비밀번호 확인. **개별 변수** 사용 시 `PG_PASSWORD` 를 문자열로 설정. **DATABASE_URL** 사용 시 비밀번호에 **특수문자(`#` 등)** 가 있으면 URL 인코딩 (예: `#` → `%23`) |
| `DATABASE_URL not set` / `DB not configured` | `.env` 파일이 로드되지 않음 (경로·이름·인코딩) | 1) `.env` 가 **server 폴더 안**에 있는지 확인 2) 파일 이름이 **`.env`** 인지 (`.env.txt` 아님) 3) **UTF-8** 로 저장. 코드에서 `path.join(__dirname, '.env')` 로 명시 로드 |
| `"sales_tool" 데이터베이스 없음` | 해당 DB 미생성 | pgAdmin에서 Databases 우클릭 → Create → Database → 이름 `sales_tool` |
| `.env` 파일 형식 | Windows에서 확장자 숨김으로 `.env.txt` 로 저장됨 | 저장 시 "모든 파일" 선택, 파일 이름을 **`.env`** 로만 지정. 또는 CMD에서 `echo ... > .env` 로 생성 |

### 3.3 .env 예시 (server 폴더)
```
# 한 줄 URL (비밀번호에 # 있으면 %23 로)
DATABASE_URL=postgresql://postgres:비밀번호@localhost:5432/sales_tool

# 또는 개별 변수
PG_HOST=localhost
PG_PORT=5432
PG_USER=postgres
PG_PASSWORD=비밀번호
PG_DATABASE=sales_tool
```

### 3.4 확인 방법
- 서버 실행: `cd server` → `npm start`
- 브라우저: http://localhost:3000 → 서버 동작 확인
- http://localhost:3000/health → `"db":"connected"` 이면 DB 연동 성공
